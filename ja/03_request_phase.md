# レビュー依頼準備のヒント

ドキュメントの準備が整ったら、次はレビューを依頼するフェーズです。
レビュアーに快く、そして効果的にレビューしてもらうためには、
依頼の仕方にいくつかのコツがあります。

## 1. 焦点を絞ったレビュー依頼の作成

- **レビュー依頼は小さく、具体的に**: 一度に大量のドキュメントを
  レビュー依頼するのではなく、章ごとや機能ごとなど、論理的な単位で
  分割して依頼しましょう。レビュアーの心理的負担が減り、レビューの質も向上します。
- **WIP (Work In Progress) の活用**: 完成してからレビューに出すのではなく、
  ある程度まとまった段階で「WIP」として共有し、早期にフィードバックを
  求めるのも有効です。手戻りが少なくなります。

## 2. 効果的なレビュー依頼文の書き方

- **何をレビューしてほしいのかを明確にする**: 「全体的に見てください」という
  漠然とした依頼ではなく、「〇〇の仕様について、技術的な観点でレビューをお願いします」
  「△△の章の構成について、分かりにくい点がないか確認してほしいです」など、
  レビューしてほしい観点を具体的に伝えましょう。
- **背景やコンテキストを伝える**: このドキュメントが作成された背景、
  解決したい課題、関連するチケットなどを依頼文に記載します。
  レビュアーが文脈を理解する助けとなります。
- **特に見てほしい点をハイライトする**: 特に自信がない部分や、議論したい点、
  重要な変更点などを具体的に示し、レビュアーの注意を促します。
- **期待する期限を伝える**: いつまでにレビューしてほしいのか、
  具体的な期限を（理由とともに）伝えましょう。ただし、相手の都合を無視した
  一方的な期限設定は避けるべきです。

## 3. セルフレビューの実践

- **レビュアーの視点で読み返す**: 依頼する前に、一度自分がレビュアーになったつもりで
  ドキュメントを客観的に読み返してみましょう。誤字脱字、分かりにくい表現、
  構成の乱れなど、多くの問題点を自分で発見できます。
- **チェックリストを活用する**: プロジェクトやチームで定められたドキュメントの
  チェックリストがあれば、それに沿ってセルフチェックを行います。

## 4. 避けるべきよくある落とし穴

- **丸投げ依頼**: 背景説明もレビュー観点の指定もなく、
  ただドキュメントのリンクを貼るだけの依頼は避けましょう。
- **レビュー依頼の放置**: 依頼した後は、レビュアーからの質問に迅速に回答し、
  議論を前に進める責任があります。
- **完璧主義**: すべてが完璧になるまでレビューに出さない、という姿勢は、
  手戻りのリスクを増大させます。適切なタイミングでフィードバックを求める
  勇気を持ちましょう。
