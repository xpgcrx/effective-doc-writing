# 付録：ドキュメントレビューチェックリスト

このチェックリストは、ドキュメントレビューを行う際の観点を網羅的に確認するためのものです。
すべての項目を常にチェックする必要はありません。ドキュメントの種類やレビューの目的に応じて、
適宜利用してください。

## 1. 構成と論理

- [ ] ドキュメントの目的と対象読者が明確に定義されているか？
- [ ] 全体像から詳細へ、または時系列など、論理的な順序で構成されているか？
- [ ] 目次や見出しが適切で、全体構造を把握しやすいか？
- [ ] 各セクションのタイトルが内容を的確に表しているか？
- [ ] 情報の重複や欠落はないか？

## 2. 正確性と網羅性

- [ ] 技術的な記述（仕様、数値、手順など）に誤りはないか？
- [ ] 前提条件、制約事項、既知の問題などが明記されているか？
- [ ] 必要な情報がすべて含まれており、読者がタスクを完了するために十分か？
- [ ] 関連するドキュメントやチケットへのリンクは正しく、切れていないか？

## 3. 明確さと分かりやすさ

- [ ] 専門用語や略語が多用されすぎていないか？（必要な場合は注釈があるか）
- [ ] 一文が長すぎず、シンプルで理解しやすいか？
- [ ] 曖昧な表現（「など」「等」の多用など）がなく、具体的な記述になっているか？
- [ ] 図、表、サンプルコードなどが効果的に使われ、理解を助けているか？
- [ ] 読者の視点に立ったとき、疑問に思う点や分かりにくい点はないか？

## 4. 一貫性とスタイル

- [ ] プロジェクトやチームで定められた用語や表記スタイルが統一されているか？
  （例：「サーバ」と「サーバー」、「ユーザ」と「ユーザー」など）
- [ ] 敬体（です・ます調）と常体（だ・である調）が混在していないか？
- [ ] フォーマット（インデント、フォント、箇条書きなど）に一貫性があるか？

## 5. その他

- [ ] 誤字・脱字はないか？
- [ ] 変更履歴は適切に記載されているか？
- [ ] ファイル名やドキュメントのタイトルは適切か？
